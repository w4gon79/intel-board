# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Intel Board, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please:

1. Email the maintainer directly through GitHub
2. Or open a private [GitHub Security Advisory](https://github.com/w4gon79/intel-board/security/advisories/new)

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

## Response Time

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 7 days
- **Fix:** Depends on severity, critical issues prioritized

## Scope

This policy covers the Intel Board application code. Out of scope:

- Third-party dependencies (report to upstream maintainers)
- API keys or credentials found in code (rotate immediately)
- Social engineering or physical attacks

## Responsible Disclosure

We ask that you:

- Give us reasonable time to fix the issue before public disclosure
- Do not access or modify other users' data
- Do not degrade service availability

We commit to:

- Not taking legal action against good-faith security research
- Crediting you in the fix (unless you prefer anonymity)
- Keeping you informed of progress
